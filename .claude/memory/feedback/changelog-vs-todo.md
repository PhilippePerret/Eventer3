**CHANGELOG** : détailler vraiment le travail — quoi, comment, pourquoi technique.

**TODO section Fait** : courte phrase, pas de commentaire entre parenthèses. Ex : `- [x] 2026-06-20 — FilePicker : bug frappe nouveau dossier`

**Why:** L'utilisateur veut pouvoir scanner le TODO rapidement. Le CHANGELOG est pour comprendre ce qui a été fait.

**How to apply:** Dans TODO/Fait, jamais de `(explication)` après le titre. Garder les détails pour CHANGELOG uniquement.
