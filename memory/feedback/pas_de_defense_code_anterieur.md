Ne jamais utiliser "ce code était déjà là avant mon intervention" / "je l'ai juste recopié" comme argument pour reporter ou éviter une correction. Le code appartient au projet, l'origine (Claude maintenant, Claude avant, peu importe) n'a aucune importance.

**Why:** Dit explicitement après avoir présenté `this.lister_id ?? this.id` comme "pré-existant, pas mon édit" en réponse à une critique — perçu comme une esquive/justification au lieu de traiter le problème. Tout le code du projet est "notre" code, le distinguo chronologique ne sert qu'à se déresponsabiliser.

**How to apply:** Face à un problème repéré dans le code (peu importe son ancienneté), le traiter directement : le signaler comme fait technique, proposer/appliquer la correction. Ne jamais mentionner qui ou quand ça a été écrit comme justification pour ne pas agir maintenant. Cf. [[pas-de-bug-mais-implementation]] pour la même logique côté "bug vs implémentation manquante".
